### アイテム別設計

#### 5-1 Jenkins セットアップ（§2.3）

- `jenkins/Dockerfile` で公式 LTS イメージをベースに起動
- `--httpListenAddress=0.0.0.0` でコンテナ内バインド（コンテナ外からは nginx 経由のみ）
- Jenkins ←→ Webhook受信サーバ間はDocker内部ネットワーク（`ai-review-net`）経由で通信
- **コンテナ外への直接公開ポートなし**（nginx コンテナのみ外向きポートを持つ）

#### 5-2 nginx リバースプロキシ（§2.3）

- WSL2 は仮想NATネットワークのため、Windows側の社内NIC IP（`192.168.x.x`）に WSL 内から直接バインドできない
- nginx は `0.0.0.0:80` でバインドし、**Windows 側で portproxy を設定**して社内NIC から WSL2 へ転送する
- `proxy_pass` で Jenkins コンテナ（内部ホスト名 `jenkins:8080`）へ転送
- `INTERNAL_IP` は Windows 側の portproxy 設定コマンドのみで使用（nginx.conf には記載しない）

**Windows 側 portproxy 設定（管理者 PowerShell で実行）：**
```powershell
netsh interface portproxy add v4tov4 `
  listenaddress=${INTERNAL_IP} listenport=80 `
  connectaddress=<WSL2のIP> connectport=80
```

#### 5-3 環境変数・シークレット管理（§9.2）

`.env.example` に以下を全列挙（値は空欄またはダミー）：

```bash
GITLAB_URL=
GITLAB_TOKEN=
GITLAB_WEBHOOK_SECRET=
JENKINS_URL=http://jenkins:8080
JENKINS_USER=
JENKINS_TOKEN=
ANTHROPIC_API_KEY=
DB_PATH=/app/db/findings.db
INTERNAL_IP=192.168.x.x   # Windows側portproxyのlistenaddressに使用
```

- `.gitignore` で `.env` を除外（実値漏洩防止）
- `docker-compose.yml` は `env_file: .env` で読み込み

#### 5-4 Docker Compose 構成（§10 #4）

| サービス名 | ベースイメージ | 役割 | 外向きポート |
|---|---|---|---|
| `webhook` | `python:3.11-slim` | FastAPI Webhook受信サーバ | なし（nginx/Jenkins経由のみ） |
| `jenkins` | カスタム（`jenkins/Dockerfile`） | CI実行 | なし |
| `nginx` | `nginx:alpine` | リバースプロキシ | `0.0.0.0:80`（Windows側portproxy経由で社内NICに公開） |

- DBはコンテナ内SQLiteファイル（`webhook` サービス内）→ `db/` ボリュームでホストにマウント
- `claude` CLI は `webhook` コンテナ内にインストール済みとする（`Dockerfile` に `RUN npm install -g @anthropic-ai/claude-code` を追加）
- 全サービスは内部ネットワーク `ai-review-net` で接続

### 実装ステップ

| # | ステップ | 対応アイテム |
|---|---|---|
| ① | `.gitignore` 作成（`.env` 除外） | 5-3 |

| ⑥ | デプロイ手順をREADMEに記載（下記手順） | 5-1〜5-4 |

**デプロイ手順（README 記載内容）：**
1. `cp .env.example .env` → 各変数に実値を記入
2. WSL2 Ubuntu 上で `docker compose up -d`
3. WSL2 の IP を確認: `ip addr show eth0 | grep 'inet '`
4. Windows 管理者 PowerShell で portproxy 設定:
   ```powershell
   netsh interface portproxy add v4tov4 listenaddress=<INTERNAL_IP> listenport=80 connectaddress=<WSL2 IP> connectport=80
   ```
5. ファイアウォール確認（必要に応じてポート 80 を許可）|
