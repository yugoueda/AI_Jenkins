### アイテム 1-3：スキーマ詳細設計（インデックス・制約）

**findings テーブル**

| 設計項目 | 内容 |
|---|---|
| CHECK制約 | `source IN ('AI')` |
| CHECK制約 | `status IN ('OPEN', 'APPLIED', 'REJECTED')` |
| NOT NULL | `mr_id`, `source`, `status`, `created_at`, `updated_at` |
| DEFAULT | `created_at`, `updated_at` = `CURRENT_TIMESTAMP` |
| 任意カラム | `fix_patch` — 指摘IDに紐づくunified diff |
| 任意カラム | `fix_patch_sha256` — 提案外変更混入防止用のパッチハッシュ |
| INDEX 1 | `idx_findings_mr_id (mr_id)` — MR単位一括取得 |
| INDEX 2 | `idx_findings_status (status)` — 状態フィルタ |
| INDEX 3 | `idx_findings_mr_id_status (mr_id, status)` — MR内状態絞り込み |

修正パッチは `.patch` ファイルとして保存せず、`fix_patch` にunified diff文字列を直接保持する。  
`fix_patch_sha256` は同一レコードで管理し、approve時の改ざん・提案外変更混入の検知に使う。

**job_queue テーブル**

| 設計項目 | 内容 |
|---|---|
| CHECK制約 | `event_type IN ('REVIEW', 'APPROVE', 'RE_REVIEW', 'UNIT_TEST_GEN')` |
| CHECK制約 | `status IN ('WAITING', 'PROCESSING', 'DONE', 'FAILED')` |
| NOT NULL | `mr_id`, `event_type`, `payload`, `status`, `created_at` |
| DEFAULT | `created_at` = `CURRENT_TIMESTAMP`, `status` = `'WAITING'` |
| INDEX 1 | `idx_job_queue_status (status)` — `is_agent_busy()` の `PROCESSING` 検索 |
| INDEX 2 | `idx_job_queue_mr_id_status (mr_id, status)` — `can_process()` の同一MR検索 |
| INDEX 3 | `idx_job_queue_created_at (created_at)` — WAITING取得のFIFO順ソート |
